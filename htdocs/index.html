このメッセージが表示されているということは、インストールが正常に完了していない可能性があります。

次の点を確認してください。

  • index.html ファイルを直接ウェブブラウザで開いているかもしれません。ウェブサーバー Apache を経由して表示してください。
  • .htaccess の設定が反映されていない可能性があります。Apache の設定で AllowOverride All などとして、設定の上書きを許可してください。
  • Apache モジュールの mod_rewrite が利用できない可能性があります。有効になるように設定する、またはモジュールのインストールを行ってください。

次のステップ

サイトマップを編集します

はじめに、サイト全体の構造をサイトマップに登録します。1行が1ページで表現されるCSV形式のファイルです。

サイトマップCSVは次のパスに置かれています。

./_PX/sitemaps/

テーマを編集します

テーマは、コンテンツエリア以外のHTMLソースを生成するPHPテンプレートです。
次のパスに置かれています。

./_PX/themes/

テーマは、複数登録して切り替えてプレビューすることができます。パラメータに ?PX=themes をつけてアクセスすると、テーマを選択する画面が開きます。

コンテンツを編集します

コンテンツは、コンテンツエリアのソースだけが記述されたシンプルなHTMLファイルです。
サイトマップの path 欄に登録したパスにHTMLファイルを作成してください。

パブリッシュします

サイトができあがったら、そのまま静的なHTMLに書き出すことができます。
パラメータに ?PX=publish をつけてアクセスすると、パブリッシュの管理画面が開きます。

あとは、ボタンを押すだけ。
しばらくすると、次のパスに生成されます。

./_PX/_sys/publish/

Pickles Framework では、 FESS が規定するコーディング規則およびドキュメントモジュール定義を利用しています。詳しくは次のページを参照してください。

  • ./_PX/sitemaps/_docs.csv

PXコマンドに関する注意事項

PXコマンドは、パブリッシュ機能(?PX=publish) や テーマ選択機能(?PX=themes)の他にも、Pickles Framework を便利に使うためのさまざまな機能を提供します。PXコマンドはサーバー内部の情報にアクセスしたり、サーバー上のデータを書き換えるインターフェイスを提供するため、第3者にアクセスされると大変キケンです。
Pickles Framework をインターネット上のサーバーで動かす場合には、次のことに注意してください。

  • ウェブ制作環境として利用する場合、利用基本認証やIP制限などの処理を施し、一般のユーザーがアクセスできない場所に設置してください。
  • または、Pickles Framework 上に構築したウェブアプリケーションをサービスとして公開する場合、設定ファイル (./_PX/configs/mainconf.ini) の [system] allow_pxcommands の値を 0 に設定し、PXコマンド機能を無効にしてください。 
    [system]
allow_pxcommands = 0 ; PX Commands の実行を許可するフラグ(1=許可, 0=不許可)
